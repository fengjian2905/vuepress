            风间 2020-03-27 用户、权限、分组 BAC JWT令牌 SpringBoot

            权限：一个api对于用户来说是否可以访问

            # 基本概念

            分组、角色
            用户
            权限

            用户不跟权限有关系， 分组跟权限有关系， 用户必须属于某个分组

            # 实现机制

            c端的权限不必设置为动态的
            @Admin @VIP分组注解不合适，稍显麻烦
            @ScopeLevel注解，权限分级，给api定一个级别，比如8， 给某一个分组定一个级别，比如VIP分组定9

            # 登录

            账号、密码

            微信登录
            账号 密码 静默登录
            code -> api -> 微信服务器

            小程序用户登录过程：

            1. code换取用户的openid
            2. user id -> uid
            3. 注册 -> openid 写入user / 查询user uid
            4. -> uid
            5. uid写入到jwt
            6. jwt->小程序

            # 票据

            传统做法：票据存储在cookie中，用户登录之后一段时间内免登录
            前后端分离的：Vue APP 小程序 令牌（一段字符串）

            JWT：可以写入一些我们需要的信息

            随机字符串 Redius或其他的缓存需要存储这个字符串对应的用户信息-> uid
            JWT：用户的信息可以写入在JWT生成的字符串中，而且JWT还具有时效性

            # 无感知二次登录

            二次登录的原因是因为令牌具有时效性，
            比如令牌有效期设置为7天，7天之后需要用户名、密码重新登录
            从用户体验上来说是不好的

            设置长令牌：

            1. 还是无法避免需要重新登录的问题
            2. 不安全，令牌丢失

            令牌有效期通常设置为：7200s

            令牌丢失无法避免，短的有效期可以提高安全性

            更安全的做法:（建立用户常用IP地址段）

            1. 校验是否是JWT，不能随便一个字符串就通过
            2. JWT是否过期
            3. 反序列化JWT，将写入JWT的信息提取出来进行验证
            4. 校验用户IP是否是常用IP地址，更严格的是只固定一个IP
            5. 如果IP改变了就要求用户重新获取令牌

            对于小程序来说登录是全程无感知的，不存在一次二次

            对于一个网站来说，没有微信的体系，要实现无感知登录， 需要使用双令牌机制：
            access_token
            refresh_token
            延长令牌有效期

               public class JwtToken{
        
       private static String jwtKey;
       private static Integer expiredTimeIn;
       private static Integer defaultScoep = 8;
       
       @Value("${fengjian.security.jwt-key}")
       private void getJwtKey(String jwtKey){
         JwtToken.jwtKey = jwtKey;
       }
       @Value("${fengjian.security.token-expired-in}")
              private void getJwtKey(Integer expiredTimeIn){
                JwtToken.expiredTimeIn = expiredTimeIn;
              }
       
       public static String makeToken(Long uid,Integer scope){
       
           return JwtToken.getToken(uid,scope);    
       }
       
       public static String makeToken(Long uid){
              
          return JwtToken.getToken(uid,JwtToken.defaultScope);    
       }
       
       private static String getToken(Long uid,Integer scope){
         
         Algorithm algorithm = Algorithm.HMAC256(JwtToken.jwtKey);
         Map<String,Object> map = JwtToken.calculateExpiredIssues();
         return  jwt.create();
                    .withClaim("uid",uid)
                    .withClaim("scope",scope)
                    .withExpiresAt(map.get("expiredTime"))
                    .withIssuedAt(map.get("now"))
                    .sign(algorithm);
         
       }
       
       private static Map<String,Date> calculateExpiredIssues(){
        
            Map<String,Date> map = new HashMap<>();
            Calendar calendar = Calendar.getInstance();
            Date now = calendar.getTime();
            calenar.add(Calendar.SECOND,JwtToken.expiredTimeIn);
            map.put("now",now);
            map.put("expiredTime",calendar.getTime());
            return map;
       }
   }

               public Optional<Map<String,Claim>> getClaims(String token){
       
       DecodedJWT decodedJWT;
       Algorithm algorithm = Algorithm.HMAC256(JwtToken.jwtKey);
       JWTVerifier jwtVerifier = JWT.require(algorithm).build();
       try{
           decodedJWT = jwtVerifier.verify(jwt);
       }catch(JWTVerificationException e){
           return Optional.empty();
       }
       return Optional.of(decodedJWT.getClaims());
   }

            截取令牌（aop）

            拦截HTTP请求
            作用：流量保护（防止无限调用、请求过滤）

            方法 filter interceptor aop
            基于 servlet spring spring
            顺序 filter interceptor aop
            选择 能不用就不用 比aop简单 概念较多（切点、切面..）

            # 拦截器逻辑

            1. 获取到请求token
            2. 验证token
            3. scope
            4. 读取API @ScopeLevel level
            5. scope ? level

            PermisssionInterceptor

               public class PermissionInterceptor extends HandlerInterceptorAdapter {
   
       @Override
       public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) throws Exception {
   
           Optional<ScopeLevel> scopeLevel = this.getScopeLevel(handler);
           if (!scopeLevel.isPresent()){
               return true;
           }
           String bearerToken = request.getHeader("Authorization");
           if (StringUtils.isBlank(bearerToken)) {
               throw new UnAuthenticatedException(10004);
           }
           if (!bearerToken.startsWith("Bearer")) {
               throw new UnAuthenticatedException(10004);
           }
           String token = bearerToken.split(" ")[1];
           Optional<Map<String, Claim>> optionalMap = JwtToken.getClaims(token);
           Map<String, Claim> map = optionalMap.orElseThrow(() -> new UnAuthenticatedException(10004));
           return this.hasPermission(scopeLevel.get(), map);
       }
   
       private boolean hasPermission(ScopeLevel scopeLevel,Map<String,Claim> map){
   
           int level = scopeLevel.value();
           int scope = map.get("scope").asInt();
           if (level>scope) {
               throw new ForbiddenException(10005);
           }
           return true;
       }
   
       @Override
       public void postHandle(HttpServletRequest request, HttpServletResponse response, Object handler, ModelAndView modelAndView) throws Exception {
           super.postHandle(request, response, handler, modelAndView);
       }
   
       @Override
       public void afterCompletion(HttpServletRequest request, HttpServletResponse response, Object handler, Exception ex) throws Exception {
           super.afterCompletion(request, response, handler, ex);
       }
   
       private Optional<ScopeLevel> getScopeLevel(Object handler){
   
           if (handler instanceof HandlerMethod) {
               HandlerMethod handlerMethod = (HandlerMethod) handler;
               ScopeLevel scopeLevel = handlerMethod.getMethod().getAnnotation(ScopeLevel.class);
               if (scopeLevel==null) {
                   return Optional.empty();
               }
               return Optional.of(scopeLevel);
           }
           return Optional.empty();
       }
   
   }
