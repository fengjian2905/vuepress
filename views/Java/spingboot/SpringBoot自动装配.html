        # SpringBoot加载SDK

        如何更灵活的将bean加入IOC容器？

        1. @Component、@Configuration
          适应与自己业务中的bean加载
        2. SpringBoot主要考虑的是加载第三方SDK，如何将第三方SDK或者库加入 IOC容器中，如redius、mongo，或者我们自己写的类库提供给别人使用

        SpringBoot启动类可以看成一个超级大的配置类，他会分别加载各个小的配置类模块 启动类的作用不限于此，还包括启动IOC容器

            @SpringBootApplication
    public class MissyouApplication {
      
        public static void main(String[] args){
            SpringBootApplication.run(MissyouApplication.class,args);
        }

        @EnableAutoConfiguration注解
        主要的作用就是配置bean，放入IOC容器，跟我们写的其他Configuration配置类的作用 并没有什么不同，只是实现这个目的的过程稍复杂些

        Configuration配置类核心作用：

        1. 部分属性写在配置文件中，允许从配置文件中修改配置的bean的相关字段的属性
        2. 将bean加入IOC容器中

        我们自己写的@Configuration很简单，容易理解， 为什么要有@EnableAutoConfiguration？

        我们自己业务的类或者bean是放在工程目录下的，在SpringBoot启动类同级包或子包中的 类都会被自动加载（@ComponentScan扫描）
        @EnableAutoConfiguration的作用主要是用来加载用maven安装的第三方依赖， 如果没有这个自动装配机制，我们如何将其加入项目的IOC容器，这些第三方包并不是 源码，是已经打包好的jar包，无法将其中的bean拷贝到项目目录下，SpringBoot就提供了 一种自动装配机制，将这些jar包中的bean加入到我们项目中的IOC容器中，以供使用

        @Import注解
        导入一个Selector

        AutoConfigurationImportSelector 一个重要的方法selectImports，这个方法就会加载我们第三方的配置类

        spring-boot-autoconfigure-2.2.2.RELEASE.jar中 META-INF目录下spring.factories文件中包含所有要加载的配置类， SpringBoot会读取这个配置文件中所有的配置类将其加入到IOC容器中

        为了避免SpringBoot启动类的干扰（@ComponentScan会自动将同级或子级包下的bean加入IOC容器）， 体现不出@EnableXXX的效果

        我们自己写一个类似启动类的程序

            @ComponentScan
    public class LOLApplication {
      
        public static void main(String[] args){
            
            ConfigurableApplicationContext context = 
                new SpringApplicationContextBuilder(LOLApplication.class)
                .run(args);
            ISkill iSkill = (ISkill)context.getBean("irelia");
            iSkill.r();
