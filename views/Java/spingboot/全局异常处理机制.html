          # 异常分类

          # 从Java语言机制分类

          Throwable

          Error 错误 操作系统级别，jvm虚拟机，一旦发生程序跑不起来
          Exception 异常 可以通过代码处理的 1/0

          CheckedException
          必须处理，否则代码编译阶段不通过，处理的方式：继续抛出异常或者捕获异常

          RuntimeException
          不是必须处理的，程序编译阶段不会检查，运行时会抛出异常

          异常 可以处理的，定义为CheckedException -> bug
          异常 无能为力的，定义为RuntimeException unchecked -> 数据库查询数据，未查询到，这种是无法处理的，找不到是正常的情况

          # 从另外一个角度分类

          已知异常

          未知异常
          未知异常：对于前端开发者和用户来讲是无意义的，可以打印到日志或者控制台

          分界点在于程序员是否主动处理了异常
