    通过new来产生一个具体的类，一旦这个类发生变化，对于所有依赖这个类的地方都 要去修改，对于复杂的项目而言，这种改动风险很高

    # Interface

    使用接口可以统一方法调用（策略模式）

        public interface ISkill{
    
        public void q();
        public void w();
        public void e();
        public void r();
    }
    
    public class Diana implements Iskill{
           
           ...
    }
    public class Irelia implements Iskill{
               
           ...

    public class Camile implements Iskill{
               
           ...

    public class Main {
                
        public static void main(String[] args) throws Exception{

    1. interface可以统一方法调用，但无法统一实例化
    2. 面向对象就是实例化对象，调用方法（完成业务逻辑）
    3. 只有一段代码中没有new的出现，才能保证一段代码的相对稳定，才能逐步实现ocp
    4. 上面的这句话只是表象，实质上一段代码如果要保持稳定，就不应该负责对象的实例化
    5. 对象实例化是不可能消除的
    6. 把对象实例化的过程，转移到其他的代码片段中
    7. 代码中总会出现不稳定，隔离这些不稳定，保证其他的代码都是稳定的
    8. 变化导致了不稳定
    9. 配置文件属于外部系统，而不属于代码本身

    # 通过反射机制消除变化

    HeroFactory中代码不稳定带原因？
    表面上是因为出现了new关键字，Factory类负责了对象的实例化， 实际上是由于软件系统中的变化导致了代码的不稳定，这里是用户选择的英雄是变化的， 也就是用户输入的英雄是变化的， 软件系统中的变化：用户操作的变化，还有一种变化是软件的需求或者技术的选择发生了变化（mysql数据库更改为oracle）

    HeroFactory中用户输入的变化为什么会导致代码的不稳定？
    用户不能直接创建一个对象，我们要根据用户输入的内容去new不同的对象，
    new对象的过程就是将用户的输入映射到代码中的对象

    通过反射可以直接将用户的输入映射到对象，消除变化

       public interface ISkill{
           
               public void q();
               public void w();
               public void e();
               public void r();
           }
           
   public class Diana implements Iskill{
               
               ...
        }
   public class Irelia implements Iskill{
               
           ...
    }
    
   public class Camile implements Iskill{
               
           ...
    }
    
    public Class HeroFactory{
        
        public static ISkill getHero(name) throws Exception{
            
            ISkill iSkill;
            Class<?> cla = Class.forName(name);
            Object obj = cla.newInstance();
            return (ISkill)obj;
        }
    }
    
    public class Main {
                        
        public static void main(String[] args) throws Exception{
            
            String name = Main.getPlayerInput();
            String classStr = "pro.fengjian."+name;
            Iskill iskill = HeroFactory.getHero(classStr);
            iskill.r();
                    
    }
    public static String getPlayerInput(){
                    
            System.out.print("Enter a hero's name");
            Scanner scanner = new Scanner(System.in);
            return scanner.nextLine();
            
    }
